一、需求背景

1.1 现状问题

当前家装行业普遍存在以下痛点：传统报价沟通成本极高，客户缺乏专业知识，不懂材料用量，容易盲目看总价去跟同行比价，最终导致高意向客户流失。销售人员每天需要花费大量时间进行无效沟通和手动计算报价。

1.2 为什么现在做？

需要一款极简的“获客型线上报价系统”，让客户只需输入房屋面积并选择材料档次，系统就能通过内置公式自动算出准确的预估总价。最后通过留手机号解锁详细清单，并自动同步线索至飞书，实现自动化获客和高效流转，解决拓客难、转化率低的问题。

二、需求目标

目标类型 描述 衡量指标 预期目标值
核心业务目标 降低前期沟通成本，通过自助报价获取精准的高意向客户线索。 留资转化率 > 15%
用户体验目标 将复杂的家装报价极简化，让完全不懂装修的小白也能在 1 分钟内操作完。 页面跳出率 < 40%
系统效率目标 收集到的线索自动同步到销售使用的飞书多维表格中。 数据同步成功率 100%

三、用户与使用场景

典型用户： 刚买房准备装修的小白业主，对装修价格没概念，害怕被坑，到处比价。

  1. 触达： 客户在微信群、朋友圈或直接收到销售发来的链接，在手机端打开。
  2. 输入： 客户看到“品诺有心，筑家有道”的品牌背书，输入自己的【建筑面积】。
  3. 体验“点菜”： 进入选材页。客户开始逐一点击四大类材料，体会价格从 0 开始一点点增加的掌控感。对于瓷砖，他体验到了先选规格再选价位的专业逻辑。
  4. 遇到“墙”： 选完后，底部悬浮条显示了一个预估区间价。他点击了【查看精确到元的详细清单】。
  5. 留资（核心）： 弹出留资框，为了看到具体明细，输入手机号解锁。
  6. 获得结果与同步： 解锁成功，展示高级的卡其色报价单，各项精确到元，并提供重新计算及保存图片功能。最下方展示了微信和抖音的二维码。同时，飞书多维表格瞬间新增一条包含明细的线索记录。

五、详细方案

以下根据功能清单，对各个模块进行一对一的详细逻辑描述，并附带右侧的原型沙盒供直接体验。

5.1 首页输入模块

功能点 详细规则与交互逻辑 原型演示 (沙盒切片)
面积输入与校验
  • 触发条件：用户进入系统首页默认展示。
  • 页面元素：顶部品牌 Slogan，中部提供唯一的“建筑面积”输入框。
  • 交互逻辑：
    • 限制输入格式：仅允许输入正数，支持最多1位小数。屏蔽乱码和负数。
    • 实时校验：未输入或输入无效时，“开启定制计算”按钮呈半透明置灰状态。
    • 一旦检测到有效数字，按钮高亮变为品牌色，允许点击。
  • 页面流转：点击高亮按钮后，平滑过渡至下一个“选材计价”视图。

5.2 选材与计价模块

功能点 详细规则与交互逻辑 原型演示 (沙盒切片)
购物车式选材
  • 分类导航：顶部吸顶展示四大类 Tab（基装、主材、定制、个性化），支持点击切换。
  • 选择逻辑：
    • 初始状态：所有材料均**无默认选中**。
    • 单选与反选：同类材料下各个档位互斥（单选）。点击已选中的档次可以**取消选中**。
  • UI 表现：选项主标题仅展示价格数字（如 ¥398），不带单位；下方小字副标题补充计费说明（如：按建筑面积推算...）。
瓷砖两步选择
  • 业务特殊性：因瓷砖需先定尺寸再定花色价格，采用两步展开法。
  • 第一步：展示三种主流规格（400x800, 800x800, 750x1500）。
  • 第二步：用户选中某规格后，动态在下方展开对应规格的价格档次选项（每种规格价格不同）。
实时预估总价
  • 展示位置：底部悬浮条（常驻）。
  • 动态逻辑：
    • 初始显示：“请选择材料”，右侧金额为 0。
    • 每次点击选项，系统根据后台公式实时累加精确总价（基于面积参数）。
  • “欲擒故纵”策略：最终渲染给用户的是一个**区间价**：[总价 × 0.9] - [总价 × 1.1]（数值会抹零取整到千位）。制造一种“预算慢慢上涨但在可控范围内”的心理感受。

5.3 留资解锁与结果模块

功能点 详细规则与交互逻辑 原型演示 (沙盒切片)
手机号留资弹窗
  • 触发：用户点击悬浮条“查看精确到元的详细清单”时弹出。
  • 校验规则：限制输入最大长度11位。利用正则 `^1[3-9]\d{9}$` 进行基本验证。
  • 防阻断设计：输入正确后“立即解锁”按钮亮起，点击即关闭弹窗，向后端发包，并立即跳转结果页（不因接口卡顿阻断前端展现）。
详细清单展示
与双二维码
  • 页面渲染：顶部卡片大字展示精准总价，背景色使用品牌主色 #8b5a2b。下方按四大类平铺列出所选材料明细与总金额。
  • 私域引流：页面滚动至最底端，提供两个区块分别放置“官方微信二维码”与“抖音主页二维码”。
重算与保存图片
  • 重新计算：清空所有本地状态（包含已选材料、面积、总价），返回第一个输入页面。
  • 保存图片：调用 html2canvas 插件，抓取当前DOM节点（含结果和二维码），生成一张海报图并触发浏览器下载，方便用户保存在手机相册。

5.4 后端数据同步模块

功能点 详细规则与交互逻辑
飞书多维表格同步
  • 触发时机：用户在留资弹窗点击“立即解锁”时触发异步请求。
  • 接口规范：向部署的云函数 /api/submit POST 提交 JSON 数据。
  • 业务流转：
    • 云函数接收数据，使用 @larksuiteoapi/node-sdk 携带飞书应用凭证。
    • 向指定的 Bitable（tblQpdAzySUYZiLF）追加记录（Record）。
    • 字段需严格匹配飞书表头（手机号、面积、各主材分类组文本等）。

七、异常与边界处理

异常场景 系统如何处理 / 提示文案
面积输入为空或非数字 “开启定制计算”按钮置灰不可点击。输入非数字字符时自动过滤。
未选择任何材料 底部悬浮条提示：“请选择材料”。总价区间为 0。
手机号不足11位或格式错误 解锁弹窗中的“立即解锁”按钮置灰不可点击。
飞书接口调用失败 / 网络异常 前端依然为客户展示最终报价清单结果页（绝不阻断客户核心体验流程），控制台记录错误日志供技术排查。

八、未来演进规划

  • v1.0 (当前)：跑通面积输入、按项选材、留资解锁、动态计价及飞书同步的核心流程。
  • v1.5 (规划中)：加入“套餐模式”，支持一键选择“经济型/舒适型/尊享型”打包方案，进一步简化操作。
  • v2.0 (规划中)：对接微信授权登录获取真实手机号，解决客户随意输入 11 位假号码的问题；支持管理端动态配置价格公式。

2. 飞书 API 数据同步字典

飞书多维表格（ID: tblQpdAzySUYZiLF）对应字段格式如下，已与后端 /api/submit 接口打通：

  • 手机号 (文本)
  • 建筑面积 (数字，保留一位小数)
  • 预估总价 (货币，保留两位小数)
  • 基装 (文本)
  • 主材编组：室内木门、厨卫金属门、瓷砖、厨卫集成吊顶、墙面饰材 (均为文本)
  • 定制编组：全屋定制柜、整体橱柜 (均为文本)
  • 个性化编组：电视背景墙、墙面木饰面、全屋踢脚线 (均为文本)
