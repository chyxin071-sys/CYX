# PM 工作流目录说明

本目录用于管理产品需求梳理的完整工作流，包含模板、工作流定义及输出目录。

---

## 目录结构

```
PM工作流/
├── README.md                           # 本说明文件
├── PRD模板.md                          # PRD参考模板（标准结构）
├── .agents/
│   └── workflows/
│       └── edu-pm-prd.md              # 完整工作流定义（AI代理使用）
├── 需求文档/                            # PRD 输出目录（按项目创建子目录）
├── 原型/                                # 原型文件输出目录
└── 流程图/                              # 流程图输出目录（Mermaid / 图片）
```

---

## 使用说明

### 1. 启动工作流

将你的需求描述发送给 AI，指定使用 `.agents/workflows/edu-pm-prd.md` 工作流：

```
请使用 PM工作流/.agents/workflows/edu-pm-prd.md 帮我梳理以下需求：
[你的需求描述]
```

### 2. 输出规范

| 输出类型 | 存放路径 | 文件命名规范 |
|----------|----------|--------------|
| PRD 文档 | `需求文档/[项目名]/` | `PRD_[功能名]_v1.0.md` |
| 原型说明 | `原型/[项目名]/` | `原型_[页面名].md` |
| 流程图 | `流程图/[项目名]/` | `流程_[功能名].md` |

### 3. PRD 模板使用

参考 `PRD模板.md` 作为文档结构基础，工作流会自动按模板填写内容。

---

## 工作流阶段

根据 `edu-pm-prd.md` 定义，工作流包含以下阶段：

1. **需求摄入** — 解析用户需求，提取核心信息
2. **信息架构** — 构建功能模块树与信息架构
3. **PRD 生成** — 按模板生成完整 PRD 文档
4. **原型描述** — 生成页面原型文字说明
5. **流程图生成** — 输出关键业务流程 Mermaid 图

---

## 注意事项

- 每个项目在 `需求文档/`、`原型/`、`流程图/` 下创建独立子目录
- PRD 文档应标注版本号，修改时递增版本
- 流程图优先使用 Mermaid 格式，确保可在文档中渲染
