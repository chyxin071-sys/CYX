品诺筑家
高端私宅 提案工作台

以沉浸式互动和模块化专业叙事，赋能门店设计师。通过优雅的数字体验，提升高净值客户的信任感与签单转化率。

体验高保真原型

01 项目概述与价值目标

1.1 项目背景与痛点

在高端私宅定制设计领域，提案环节是达成交易的核心节点。然而，目前门店在实际业务中普遍面临以下痛点：

  • 设计师“会做图但不会讲”： 门店设计师普遍存在渲染图做得很漂亮，但在向客户提案时缺乏逻辑和感染力，无法有效传达设计价值，导致转化率低下，容易丢单。
  • 展示形式单一枯燥： 传统提案往往是拿着几张静态效果图、全景图或枯燥的 PPT 直接给客户看，客户像看幻灯片一样，缺乏互动感和代入感，无法产生共鸣。
  • 前后逻辑割裂： 提案时，户型改造痛点、设计灵感（Moodboard）、最终效果图往往分散在不同软件（如 CAD、图片浏览器）或文件夹里，讲解过程不连贯，削弱了专业度。

1.2 产品核心目标

打造一款专为高端私宅定制门店量身定制的“提案提词器与沉浸式展示工具”。通过规范化的模块框架引导设计师梳理设计逻辑，通过极具视觉张力的“沉浸互动模式”向高净值客户传递设计的尊贵感与专业度。

1.3 商业价值 (Business Value)

  • 提升签单转化率： 通过结构化、故事化的提案流程，放大设计方案的闪光点，直接拉升客户的信任感，缩短决策周期。
  • 沉淀品牌资产： 改变过去提案资料散落在设计师个人电脑的局面，将所有优秀提案云端化、结构化沉淀，形成门店的数字资产库和新人培训教材。
  • 降本增效： 极简的工作台让设计师只需“传图+填空”即可生成百万级视觉效果的提案，免去制作复杂 PPT 的时间成本，专注设计本身。

1.4 用户价值 (User Value)

  • 对门店设计师： 获得了一个强大的“提案辅助大脑”。内置的专业文案模板和结构化排版，帮助他们自信地讲述设计故事，摆脱“茶壶里煮饺子”的窘境。
  • 对高净值客户： 获得“所见即所得”的尊贵服务体验。不再是被动看图，而是跟随设计师的节奏，在沉浸式的细节互动（Pins）中感受设计的用心与家的未来样貌。
  • 对门店管理者： 获得了一个全局把控方案质量的抓手。通过超管视角，随时抽查方案水准，确保门店对外输出的专业度高度统一。

02 用户角色与场景

2.1 核心角色分析

  • 门店设计师 (普通员工)： 具备优秀渲染能力，但口头表达和逻辑梳理能力参差不齐。需要一个极其简单的工具把图片和文案组合起来，并依赖专业的文案模板作为“提词器”。
  • 门店主管 / 老板 (超级管理员)： 负责把控门店整体设计产出质量，需要审核设计师的方案。需要方便地查看全公司所有设计师做好的提案，确保优秀方案作为资产沉淀在云端。
  • 高净值客户 (最终受众)： 对审美和细节要求极高，时间宝贵，注重体验和专属服务感。希望看到直观、连贯、有格调的家装愿景呈现，希望设计师能讲清楚每一个设计亮点背后的考量。

2.2 核心使用场景

  • 场景一：方案筹备期（工作台模式）
    设计师在自己电脑上输入员工码登录。上传客餐厅效果图，系统自动生成专业文案模板。设计师填入设计理念，并在图片上打上“隐形收纳”等亮点标签，点击“保存”同步到云端。
  • 场景二：线下门店提案（沉浸提案模式）
    客户来到门店，设计师将网页投屏。切换至“沉浸提案”模式。界面纯净化，设计师一边展示高清大图，一边对照左侧的专业文案讲解。点击图片上的“亮点标记”，右侧丝滑弹出细节说明，互动感极佳。

04 详细功能描述

本章节与《03 需求功能清单》严格一一对应，详细定义每个功能点的交互规则与界面表现。每个功能均包含多状态的原型截图，以完整呈现交互链路。

1.1 暗号静默登录

触发条件： 用户首次访问页面，或本地 localStorage 缓存失效时。

交互逻辑： 采用全屏毛玻璃遮罩覆盖页面。用户需输入系统预设的“内部员工码”（如 PN1001）。输入框支持回车键快捷提交。系统会自动对输入进行格式化（全角转半角、去空格、大写）。若输入错误，输入框产生左右抖动反馈；输入正确则丝滑隐藏遮罩，进入工作台。

状态 1：全屏毛玻璃登录界面

状态 2：输入暗号状态

状态 3：输入错误抖动提示

1.2 数据隔离与角色

触发条件： 员工码验证通过，进入主界面初始化数据时。

交互逻辑： 登录后，左上角显示当前员工码及角色标识（普通员工 / 超级管理员）。系统会根据角色标识发起不同的云数据库查询请求。普通员工只能查看并上传属于自己的方案；超级管理员（PN8888 / PN9999）拥有全局视角，拉取数据时获取全公司历史库。

状态 1：普通员工视图 (左上角仅显示员工码)

状态 2：超级管理员视图 (带特殊颜色标识)

1.3 退出与自动保存

触发条件： 用户点击左上角账号 LOGO 触发下拉菜单，并选择“退出账号”。

交互逻辑： 点击“退出”后，页面弹出居中优雅确认框。点击确认后，系统会拦截退出动作，强制优先静默执行云端保存或本地缓存。只有在确保当前空间的最新修改不丢失（收到保存成功的 Promise 返回）后，才清除本地授权记录，并退回全屏登录界面。

状态 1：点击左上角弹出的下拉菜单 (包含退出选项)

状态 2：退出二次确认弹窗

1.4 项目方案管理

触发条件： 用户在侧边栏点击“新增方案”或编辑当前方案信息。

交互逻辑： 支持新建空白方案。对于当前方案，可以点击标题区域进行编辑（如客户名称、项目地址等基本信息）。同时支持删除不再需要的历史方案（二次确认防止误删）。

状态 1：编辑项目基本信息

状态 2：新增或删除方案

2.1 多空间管理

触发条件： 用户点击顶部居中的 Tabs 区域进行空间切换或点击“+”号新增空间。

交互逻辑： 顶部居中位置的 Tabs 控制器。点击“+”号会弹出新增空间窗口，支持输入新的空间名称（如：客餐厅、主卧、厨房等）。每个空间的数据相互独立。当空间数量过多超出容器宽度时，隐藏原生滚动条，支持鼠标悬停时使用滚轮横向平滑滚动，确保不挤压右侧全局操作按钮。

状态 1：点击+号弹出新增空间窗口

状态 2：空间过多时，横向滚轮滑动效果

2.2 自定义模块与排序

功能说明：

  • 允许设计师根据不同方案的需求，灵活调整左侧文案区域的模块结构。

交互逻辑：

  • 点击“添加模块”按钮，在左侧区域底部新增一个空白的自定义内容块。
  • 鼠标按住模块右上角的“拖拽”图标，可上下拖动改变模块的显示顺序。
  • 点击模块右上角的“删除”图标，二次确认后移除该模块及内部文字。

状态 1：点击添加自定义模块

状态 2：拖拽调整模块顺序

状态 3：二次确认删除当前模块

2.3 核心材质录入

功能说明：

  • 结构化地展示方案中使用的关键材质，提升提案的专业度和客户的感知。

交互逻辑：

  • 在指定的材质模块中，点击输入框即可输入材质名称（如：微水泥、胡桃木、艺术涂料等）。
  • 支持修改已录入的材质名称。
  • 点击图片占位符支持上传对应的真实材质参考图。
  • 数据实时保存，并在沉浸提案模式下以精美的 UI 卡片形式呈现给客户。

状态 1：添加并输入核心材质名称

状态 2：修改材质名称

状态 3：上传材质参考图片

2.4 品牌颜色绑定

功能说明：

  • 为空间或材质提取核心主题色，建立直观的色彩感知体系。

交互逻辑：

  • 点击添加颜色模块，自动生成空白颜色卡片。
  • 点击色块调出系统原生取色器（Color Picker），可以在屏幕上吸取颜色，或手动输入 HEX/RGB 色值。
  • 支持点击颜色卡片下方的文本框，修改并自定义该颜色的名称描述。

状态 1：添加颜色模块

状态 2：调出系统原生取色器

状态 3：修改颜色名称

2.5 多图轮播与管理

功能说明：

  • 管理当前空间下的多张效果图，支持追加、切换与删除。

交互逻辑：

  • 点击右侧主视觉区的上传按钮，支持多张高清图批量上传。
  • 点击“+ 追加图片”按钮，可在现有图库基础上继续添加图片。
  • 底部生成毛玻璃缩略图导航，点击缩略图或左右箭头可无缝切换主图。
  • 点击右上角“垃圾桶”图标可单独删除当前展示的图片，系统会自动清理该图片关联的所有打点数据。

状态 1：初始上传界面

状态 2：正常默认空间展示

状态 3：多图轮播与缩略图

状态 4：删除当前展示图片

3.1 双模式与布局切换

触发条件： 点击顶部导航栏右侧的 Toggle 开关。

交互逻辑： 系统提供双重状态控制：
— 状态切换：“工作台”（显示所有边框、占位符、上传按钮，供设计师编辑） vs “沉浸提案”（隐藏所有编辑类元素，进入纯净阅读态，供向客户展示）。
— 布局切换：“标准”（左文右图，经典平分） vs “沉浸”（图片默认占满全屏，文字隐藏，点击亮点时再呼出文本）。

状态 1：工作台模式 + 标准布局

状态 2：沉浸提案模式 + 标准布局

状态 3：沉浸提案模式 + 沉浸布局

3.2 亮点打点 (Pins)

功能说明：

  • 在效果图上标记设计亮点，支持自定义描述文本，并提供增删改查完整闭环。

交互逻辑：

  • 在工作台模式下开启“细节解读”开关，鼠标点击主效果图任意位置生成白色圆点坐标。
  • 系统自动计算基于图片宽高的相对百分比（保证缩放响应式）。
  • 按住点位可拖拽微调坐标。
  • 点击已生成的点位，会弹出一个独立的输入浮层，输入框与点位拖拽解耦，彻底解决拖拽与点击事件的冲突。
  • 在选中点位或展开详情时，可以点击右上角的删除图标，永久移除该亮点。

状态 1：点击生成脉冲圆点

状态 2：弹出的悬浮文本输入框

状态 3：多点位打点效果

状态 4：删除指定的亮点

3.3 细节解读面板与分屏控制

功能说明：

  • 提供沉浸式的细节解读体验，支持用户自定义拖拽分屏比例。

交互逻辑：

  • 在“沉浸提案”布局下点击效果图上的亮点，主图按比例缩小并整体向左侧移动，右侧丝滑滑出详细图文解读抽屉。
  • 中间会出现一个拖拽控制条（Resizer）。
  • 鼠标按住控制条左右拖动，可以实时动态改变左侧主图和右侧详情面板的宽度比例，松开后自动固定。

状态 1：点击 Pin 点呼出面板

状态 2：拖拽分屏控制条前

状态 3：拖动改变宽度比例

4.1 云开发平台对接

触发条件： 用户完成编辑，点击右上角“保存”按钮。

交互逻辑： 采用无后端架构，前端引入 Tencent CloudBase SDK。点击保存后，按钮转为 Loading 状态并锁定防连点。系统首先将本地的 Base64 缩略图还原为 File 对象，调用云存储 API 上传，获取公网 CDN 链接。随后将界面所有的文本内容、坐标数组、图片链接整合成 JSON 对象，调用云数据库 API 进行写入。完成后恢复 UI 并弹出“保存成功”提示。

状态 1：点击保存，按钮转为 Loading (正在同步至云端)

状态 2：保存成功，右上角弹出优雅的 Toast 提示

                        M --> N[提取当前空间文案、标题、Pins数据]
                        N --> O[调用 db.collection.add 写入云数据库]
                        O --> P[恢复 UI 状态, 提示保存成功]

06 数据埋点与异常处理

6.1 核心数据埋点需求

为了后续分析转化率和优化产品，系统需在关键链路进行数据埋点：

事件名称 触发时机 上报参数 (Properties)
login_success 用户成功输入暗号进入工作台 employee_code, role (normal/admin), timestamp
image_uploaded 本地图片成功加载至浏览器 image_count, file_size_mb
pin_created 在图片上成功添加一个新的互动打点 space_id, pin_coordinates, has_text (boolean)
mode_switched 切换顶部“沉浸提案”或“细节解读”开关 target_mode (present/edit/focus), duration_in_prev_mode_sec
cloud_save_success 提案成功写入云数据库 total_pins, text_length, is_update (boolean)

6.2 边界与异常处理 (Edge Cases)

  • 断网/弱网环境： 用户在编辑时断网，系统通过 localStorage 实时保存本地。点击“保存云端”时若失败，抛出优雅的 Toast 提示“网络异常，已为您保存在本地，请稍后重试”，不丢失任何数据。
  • 超大图片上传： 前端读取本地图片时，若单张图片超过 10MB，自动在前端进行 Canvas 压缩，保证 Base64 字符串不会导致浏览器内存溢出（OOM）。
  • 登录态过期： 员工码被后台封禁或更改。用户点击保存时，若云端接口返回 401 Unauthorized，拦截保存动作，弹出重新登录遮罩，登录成功后自动接续之前的保存动作。
  • 极端比例图片： 客户上传极宽或极长的长图。容器采用 object-fit: cover/contain 自适应，确保 Pin 点的百分比坐标 (x%, y%) 在任何屏幕尺寸下计算都不会发生偏移。

07 未来演进规划 (AI 赋能)

在 V2.0 阶段，工作台将从“辅助提词器”向“智能生成大脑”演进，全面接入大语言模型 (LLM) 和视觉多模态大模型 (VLM) 能力：

  • 图生文 (Image to Text) 智能解析：
    设计师上传效果图后，不再需要手动填空。系统自动调用视觉大模型（如 GPT-4V）分析图中材质（如：微水泥、木格栅）、风格（如：极简诧寂）、灯光（如：无主灯氛围），一键生成高级定制文案。
  • AI 智能打点识别：
    系统自动识别图中的核心家具和结构亮点（比如自动识别出“隐形门”、“悬浮电视柜”），并自动在相应坐标生成 Pin 点建议，设计师只需点击确认即可。
  • 客户意向分析：
    在“沉浸提案”模式下，记录客户对哪个 Pin 点停留时间最长、展开次数最多，提案结束后自动生成《客户意向热力图报告》发给设计师，助力精准逼单。
