一、问题定义与边界

1.1 组织与沟通现状

  • 企业规模： 20–30 人的传统家装公司，组织精简。
  • 部门结构： 设计部、销售部、项目部、财务部。
  • 协作方式： 以微信群为主，通知、图片、文件分散，易丢失、难追溯。
  • 现场到信息链： 现场→项目经理拍照→微信群→业主/设计协同，缺乏标准化与留痕。

1.2 待解决的关键问题

  • 信息碎片化： 微信对话沉底，关键节点与文件难以检索与复用。
  • 进度不透明： 业主无清晰节点与预计完工时间，焦虑感强。
  • 材料环节纠纷： 型号、数量、时点佐证不足，易引发争议。
  • 多工地管控难： 项目经理跨工地排期冲突与延期联动管理缺失。

1.3 渠道选择（为何小程序）

  • 触达成本低： 团队与业主已在微信生态，免安装、免学习成本。
  • 社群联动： 与微信群天然联动，系统通知可“同步到群”。
  • 移动现场友好： 轻量、弱网友好，利于拍照上传与断点续传。
  • 可渐进扩展： 先跑闭环，后扩 H5/后台管理。

1.4 角色与职责

角色 核心目标 主要任务 关键指标
业主 随时掌握进度，放心省心 查看日志、确认材料、反馈问题 日志阅读率、反馈响应时效
项目经理/工长 多工地统筹，标准化交付 发布日报、排期联动、材料核验 日报覆盖率、延期率、处理闭环率
设计师 效果还原与变更控制 节点图纸与变更说明对齐 变更次数、复工率
销售 转化与口碑 跟单、引导使用“成长播报” 转介绍率
财务 里程碑节点对账 按合同节点跟进收款提醒 回款及时率

二、目标与衡量

2.1 目标体系

目标类型 描述 衡量指标 目标值
核心目标 建立标准化的在线施工播报与进度追踪体系，让业主能实时查看新家变化 业主端周活率 (WAU) > 80%
核心目标 提升项目经理多工地管理效率，规范化资料与排期流转 单项目平均线上沟通耗时缩减 降低 30%
次要目标 利用邀请码机制，将“装修日记”辐射至业主家庭其他成员，形成私域资产 单项目平均绑定业主数 ≥ 2.5人

2.2 成功验收标准与边界

  • 日报覆盖率： 核心在建项目日报覆盖 ≥ 90%
  • 反馈闭环： 业主反馈 24 小时内闭环 ≥ 95%
  • 材料确认： 线上确认率 ≥ 80%，异常分支有闭环记录
  • 非目标： 不含 ERP 全量财务/采购，不做商城交易闭环

2.3 用户价值

  • 业主： 随时查看新家进度，减少焦虑，重要决策有据可查。
  • 员工： 多工地统筹，排期与材料节点标准化，减少沟通损耗。

四、详细方案

4.1 公共基础模块

4.1.1 身份选择与验证 P0

  • 功能说明
    选择“业主”或“员工”身份，并输入对应的邀请码或管理口令进入项目。
  • 1. 触发条件
    用户首次打开应用，或退出登录后。
  • 2. 展示规则
    • 顶部展示品牌Slogan与Logo。
    • 下方并排展示“我是业主”和“我是管理者”两个入口卡片。
  • 3. 交互逻辑
    flowchart LR A["选择身份: 业主/员工"] --> B["弹出邀请码/管理码输入层"] B --> C{"输入是否有效"} C -- 有效 --> D{"身份类型"} D -- 业主 --> E["校验邀请码有效期/绑定次数"] D -- 员工 --> F["校验员工口令与企业权限"] E --> G["写入会话态并跳转业主首页"] F --> H["写入会话态并跳转工作台"] C -- 无效 --> I["标红输入框 + Toast 原因"] I --> B G --> J{"首次访问?"} J -- 是 --> K["引导关注项目群/开启通知"] J -- 否 --> L["进入首页"]
  • 4. 状态变化
    • 默认状态：展示角色选择页面。
    • 触发后状态：显示输入键盘及半屏模态框。
    • 结束状态：成功进入项目详情/工作台页面。
[!IMPORTANT]
⚠️ 注意：业主邀请码按工地分配（一户一码），员工管理码仅限内部人员知晓，防范隐私泄露。

4.2 员工端：管理工作台

4.2.1 工地监控大盘 P0

  • 功能说明
    总览所有在建工地，支持按“在建项目、需关注、全部项目”快速筛选。显示项目名、当前阶段、地址、开工日期及进度条。
  • 1. 触发条件
    管理者成功登录后默认进入的主工作台页面。
  • 2. 展示规则
    • 顶部为数据统计概览及分类Tab。
    • 下方列表按项目开工时间倒序排列。
    • 空状态展示“暂无项目”插图与“新建项目”按钮。
    • 网络异常时在列表区域展示重试提示和按钮。
    • 权限变更时拦截进入并提示重新登录/更换身份。
  • 3. 交互逻辑
    flowchart LR A["点击筛选 Tab"] --> B["更新查询条件"]; B --> C["发起接口请求 (debounce 300ms)"]; C --> D{"返回是否为空"}; D -->|否| E["渲染列表并计算进度条"]; D -->|是| F["显示空状态 + 新建引导"]; E --> G["点击项目卡片"]; G --> H{"权限校验"}; H -->|通过| I["跳转项目详情面板"]; H -->|拒绝| J["提示权限变更，返回登录"]; C --> X{"请求失败?"}; X -->|是| Y["提示网络异常/重试"]; F --> Z["点击新建项目"]; Z --> N1["跳转建档表单"];
  • 4. 状态变化
    • 默认状态：显示全部在建项目。
    • 触发后状态：显示特定筛选结果。
    • 结束状态：进入特定项目内页。
[!IMPORTANT]
⚠️ 进度条数据需实时读取工序节点的完成状态。

4.2.2 新工地建档 P0

  • 功能说明
    录入项目地址、客户信息、房屋面积与开工日期，快速初始化数字化施工档案。并支持批量导入施工节点。
  • 1. 触发条件
    管理者在工作台点击“新建项目”按钮。
  • 2. 展示规则
    • 分步式或长表单形式展示输入项，必填项需标红。
    • 提交按钮固定在屏幕底部。
  • 3. 交互逻辑
    flowchart LR A["填写表单"] --> B["点击提交"] B --> C{"系统校验字段"} C -- 成功 --> D["创建档案并跳转至新项目首页"] C -- 失败 --> E["提示必填项未完成"]
  • 4. 状态变化
    • 默认状态：空白表单。
    • 触发后状态：表单校验提示。
    • 结束状态：建档完成。
[!IMPORTANT]
⚠️ 批量导入施工节点需自动根据开工日期推算后续时间。

4.2.3 结构化施工日报 P0

  • 功能说明
    选择工序节点自动匹配标准化验收文案，支持现场拍照上传。
  • 1. 触发条件
    在项目详情页点击“写汇报”或“验收汇报”。
  • 2. 展示规则
    • 顶部选择相关工序标签。
    • 自动填充相关施工标准与验收结果模板。
    • 底部提供图片上传九宫格。
    • 上传前对图片进行本地压缩并叠加时间水印。
    • 支持草稿保存，未发布的内容在入口处展示“草稿”标识。
  • 3. 交互逻辑
    flowchart LR A["选择工序标签"] --> B["自动插入标准模板"]; B --> C["补充文本说明"]; C --> D["添加图片 (本地压缩/时间水印)"]; D --> E["点击发布"]; E --> F{"表单校验"}; F -->|通过| G["直传 OSS 获取 URL"]; G --> H["保存日志并触发推送"]; H --> I["返回动态并定位到新日志"]; F -->|不通过| J["标红缺失项并提示原因"]; G --> K{"上传失败?"}; K -->|是| L["保存草稿并提示重试"]; L --> E;
  • 4. 状态变化
    • 默认状态：空表单。
    • 触发后状态：自动填充文本，显示已选图片。
    • 结束状态：发布成功并返回首页。
[!IMPORTANT]
⚠️ 图片必须支持水印时间戳，防止后期纠纷。

4.2.4 进度计划表 P0

  • 功能说明
    针对不同需求手动增删工序，调整单个节点工期后后续计划自动顺延。
  • 1. 触发条件
    在项目详情页点击“进度管理”进入计划表。
  • 2. 展示规则
    • 按时间轴顺序排列所有节点。
    • 显示每个节点的开始、结束时间及状态（未开始、进行中、已延期）。
  • 3. 交互逻辑
    flowchart LR A["点击节点日期"] --> B["唤起日期选择器"] B --> C["选择新日期"] C --> D{"是否早于前置?"} D -- 是 --> E["阻止提交, 提示需先调整前置"] D -- 否 --> F{"是否跨越后置?"} F -- 是 --> G["弹窗: 是否联动顺延后续"] G -- 确认 --> H["重新计算后续开始/结束"] G -- 取消 --> I["仅更新当前, 标记风险"] F -- 否 --> J["直接更新当前节点"] H --> K["保存成功, 重绘甘特"] I --> K J --> K
  • 4. 状态变化
    • 默认状态：展示当前进度。
    • 触发后状态：显示日期选择器及冲突弹窗。
    • 结束状态：保存成功，整体计划更新。
[!IMPORTANT]
⚠️ 自动顺延逻辑：后续节点的开始时间 = 前置节点的结束时间 + 1。

4.2.5 主材到场管控 P1

  • 功能说明
    登记主材到场时间并拍照核验，确保材料品牌、型号与合同一致。
  • 1. 触发条件
    项目详情页点击“材料清单”。
  • 2. 展示规则
    • 列表展示各项材料的当前状态（待进场、已进场）。
    • 每项材料显示预定品牌、型号规格。
    • 进入核验前先行校验型号/清晰度，异常项标红并提示。
    • 业主确认状态在卡片上以标签形式可见（已确认/驳回）。
  • 3. 交互逻辑
    flowchart LR A["点击“确认进场”"] --> B["弹出上传实拍图对话框"] B --> C["上传图片并提交"] C --> D{"图片/型号校验"} D -- 通过 --> E["状态变更为“已进场”"] E --> F["同步通知业主"] D -- 不通过 --> G["标记异常并退回补充"] F --> H{"业主确认?"} H -- 确认 --> I["闭环, 进入安装待办"] H -- 驳回 --> J["异常: 要求复检/退换"]
  • 4. 状态变化
    • 默认状态：待进场列表。
    • 触发后状态：核验表单页面。
    • 结束状态：已进场，带实拍图存证。

4.2.6 反馈处理中心 P1

  • 功能说明
    集中接收并回复业主在施工播报下的咨询与反馈，确保沟通有迹可循。
  • 1. 触发条件
    工作台底栏点击“消息/反馈”。
  • 2. 展示规则
    • 按未读消息和处理状态分类。
    • 每条消息附带其对应的施工日志上下文。
    • 支持“转派负责人/挂起/已解决”状态切换与筛选。
  • 3. 交互逻辑
    flowchart LR A["点击消息"] --> B["展开详情"] B --> C["输入文字并回复"] B --> D["转派负责人"] B --> E["标记为挂起/待业主反馈"] C --> F["标记已解决"] F --> G["消息归档"]
  • 4. 状态变化
    • 默认状态：未处理列表。
    • 触发后状态：对话窗口。
    • 结束状态：消息标记为已处理。

4.3 业主端：云监工平台

10.2 工艺做法与验收标准库

注：APP已原生内置“常规汇报框架”（如自动生成汇报标题、问候语及结尾），项目经理不再需要手打模板，仅需在对应节点勾选以下具体的工艺规范及验收卡项即可：

一、杂工阶段 (开工进场 / 墙地固 / 砸墙)

【开工】

  • 设施排查：防盗门正反面完好无缺，拉手和锁运行正常。
  • 可视对讲及挂件 个
  • 开工大吉保护膜：对入户防盗门进行保护，防止防盗门被刮花、刮伤，若是木门用无纺布经行保护，防止后期掉漆。
  • 空鼓：客厅顶面、墙面，卧室顶面、墙面有空鼓现象，已做标记，需要提前告知物业公司进行处理、报备。
  • 规范要求：卧室，客厅，餐厅原始墙面有基础施工洞口，已经发现多处裂纹，我已经标记好了，后期力工进场会按照我们施工标准进行开槽处理。
  • 设施排查：原有卧室窗户完好无损，拉手合页正常。
  • 设施排查：阳台防护栏牢固，没有发现不正常松动。
  • 设施排查：煤气表1个，完好，未开通煤气。
  • 设施排查：购房配置开关0个，通电测试均完好无损。
  • 设施排查：购房配置插座0个，通电测试均完好无损。
  • 设施排查：购房配置灯泡0个，通电测试均完好无损。
  • 设施排查：购房配置配电箱1个，购房配置弱电箱1个，通电测试均完好无损。
  • 设施排查：等电位箱0个，完好无损。
  • 设施排查：空调孔0个，正常。
  • 设施排查：主次卫排水管经测试水流通畅，无堵塞现象。
  • 设施排查：厨房烟道经检查通风正常。
  • 规范要求：主次卫生间经检查没有发现楼下楼上有漏水痕迹,阳台顶面及地面经检查无漏水痕迹。
  • 临时蹲便器：工地工人临时专用。
  • 燃气表保护套：在施工中对燃气表以及燃气管进行保护。
  • 下水管封：施工中有效防止沙子水泥杂物掉入下水管道。
  • 厨房卫生间门洞大小

【墙地固】

  • 铲大白：1.原始工程腻子环保系数较差，2.便于墙面找平，原始墙面可能高低不平，铲除腻子便于石膏找平，提高施工标准。3.防止墙面附着力低，从而造成刮腻子和基层墙体结合不牢，容易空鼓或腻子脱落。
  • 黄色墙固：墙面加固处理剂是采用聚合物乳液和其他有机聚合物添加助剂精心配制而成，该产品为浅黄色液体，具有优异的渗透性，能充分浸润墙面基层，使基层坚固，增强光滑基面的附着力，粘接强度高，可营造出一个安全放心的墙面基层，可极大程度上避免墙面涂料层出现质量问题。
  • 绿色地固：强力型地固是一种专门作用于水泥地面上的涂料，适用于家庭装修或工程装修初期水泥地面的封闭处理，防止跑沙现象。地固耐水防潮，也可以避免木地板受潮气侵蚀而产生的变形，同时避免日后从地板缝隙中“扑灰”。该产品可有多种颜色，主要以绿色，蓝色，红色为主，涂刷在地面上具有颜色鲜艳，色彩分布均匀，遮盖力强等特点，干燥后不掉粉，不掉色，可随意清扫。

【砸墙】

  • 打墙：设计定位，确认位置
  • 打整墙：由下之上开始打，打之前需注意做好安全防护，以免上墙突然掉下伤人。
  • 打半边墙：确认位置，弹线，用切割机开槽至墙的一半，再打墙，可以确保打完之后剖面的平整。
  • 打墙遇到电线水管时的注意：应小心处理，妥善解决，对原有材料进行适当保护。

  • 地面保护膜：对瓷砖表面起到保护作用，防止防刮花、刮伤（主材安装后、竣工前清理地面保护膜及保洁）。
  • 铺贴完后的墙砖和地砖对空鼓方面的质量要求：①无整块空鼓；②单块砖边或角空鼓数量不超过铺贴砖总数量的5%。
  • 对卫生间铺贴完地砖后的坡度控制要求：须满足2%的排水要求。
五、木作阶段 (木工作业 / 验收)

【木工】

  • 规范要求：木工进场，设计师再次讲解木工注意，进行现场交底。
  • 大芯板：也称细木工板，是利用天然旋切单板与实木拼板经涂胶、热压而成的板材，选择细木工板一定要注意选择符合E0级以上环保标准的。夹芯板主要用在一些特殊造型，楼梯踏步等基础垫层。
  • 奥松板：奥松板，一种进口的中密度板，是夹芯板、刨花板的替代升级产品，特性是更加环保。奥松板用辐射松原木制成，主要使用原生林树木，能够更直接地确保所用纤维线的连续性奥松板一般被广泛用于装饰，家具，建筑，包装等行业，其硬度大，密度高，适合护墙板、软包、硬包、镜片等室内装饰基层。防火防潮性能优于传统大芯板、不易变形且非常环保。
  • 石膏板：采用世界500强的北新建材龙牌普通纸面石膏板，采用高性能护面纸与建筑石膏及特殊添加剂等芯材牢固粘结成型，正面呈灰白色，严格按照 GB/T 9775-1999 生产。强度高、韧性好:采用高性能护面纸，确保隔墙吊顶的强度。握钉力强:发泡率适中，单位面积重量适中，提高施工速度，降低劳动强度。护面纸粘结牢固:工艺、配方先进，不脱纸。
  • 双饰面板：也称免漆板，常用于家具制作的板材。板子表面的那层装饰纸，在学术上称三聚氢氨纸，这种纸有一定的抗打击能力，可以印刷各种图案，家具表面的木材图案就是印刷在这种专业纸上，而实际上内部的板材是刨花板。刨花板具有握钉力好，结实耐用的优点，选择双饰面板要选择符合环保E0级标准的。
  • 木方：装修中常用的一种材料,有多种型号,用于撑起外面的装饰板，起支架作用。
  • 轻钢龙骨：根据其型号、规格及用途的不同，有T形、C形、U形、卡式龙骨、L型边骨及专用配套附件等，轻钢龙骨广泛用于室内装修吊顶、隔墙、造型等。它坚固、耐用、抗腐蚀性强，在价格方面也广受建设单位和施工单位的欢迎，减少工程在施工过程中对木龙骨变形引起的质量的担忧，以及因变形而导致的频繁维修，大大降低工程施工成本和使用时因维修造成的各种烦恼。
  • 自攻螺丝：自攻螺丝是一种带有钻头的螺丝，自攻螺丝主要用于一些较薄板件的连接与固定，如夹心板，奥松板，指接板等相互的连接，轻钢龙骨边骨与副骨的连接等。优点硬度高，其穿透能力强，不易弯曲并且自身有很强的耐腐蚀能力。
  • 膨胀螺丝：一般说的是金属膨胀螺丝，膨胀螺丝的固定是利用楔形斜度来促使膨胀产生摩擦握裹力，达到固定效果。螺钉一头是螺纹，一头有锥度，外面包一铁皮（有的是钢管），铁皮圆筒（钢管）一半有若干切口，把它们一起塞进墙上打好的洞里，然后锁螺母，螺母把螺钉往外拉，将锥度拉入铁皮圆筒，铁皮圆筒被涨开，于是紧紧固定在墙上。
  • 封石膏板：窗帘盒已封石膏板，天花转角处的石膏板裁∟型固定，中间丁字接口裁T型固定，石膏板面层周边螺丝间距≤20cm，石膏板缝隙留0.5cm~1cm的V型间隔。
  • 规范要求：v形槽，石膏板接缝采用v形工艺，后期充分填补石膏更牢固，降低开裂几率。
  • 防锈自攻丝，避免后期生锈
  • 规范要求：主龙骨间距≤700，主副龙骨采用新型卡式龙骨，有一定的起拱量。
  • 规范要求：中央空调出风口采用独家细木工侧板加固，保证后期风口百叶安装牢固性。
  • 规范要求：吊顶拐角处采用品诺筑家独家护角神器，连接两根龙骨，使其成为一个整体，更加牢固，增加其稳定性，能够有效地防止后期出现拐角开裂现象。
  • 工艺要求是：石膏板饰面安装质量标准：①整体水平度＜3mm，②接缝高低≤2mm，表面无污染，无缺破坏。
  • 规范要求：边龙骨固定点间距为≤300mm，吊杆固定间距为≤600mm。
  • 规范要求：副骨排列间距≤350mm，专用自攻钉固定。
  • 规范要求：采用8mm吊杆，膨胀拉爆固定。吊杆距墙边距离≤300mm，吊杆距离主龙骨末端≤100mm。
  • 石膏板饰面安装质量标准：①整体水平度＜3mm，②接缝高低≤2mm，表面无污染，无缺破坏。

【木工验收】

  • 顶内为轻钢龙骨架
  • 规范要求：石膏板与石膏板之间留缝大于5mm,V字形
  • 吊顶转角龙骨使用斜拉工艺
  • 石膏板底板托侧板拼接工艺
  • 规范要求：转角处采用45°对角工艺，相比较转角处采用L型拼板，更有效降低应力释放
  • 规范要求：阴阳角是否垂直（允许偏差2mm）
  • 卫生间墙地砖通缝
  • 所有砖面颜色一致，无破损。
  • 地面保护膜：对瓷砖表面起到保护作用，防止防刮花、刮伤（主材安装后、竣工前清理地面保护膜及保洁）。
六、油工阶段 (乳胶漆 / 验收)

【乳胶漆项】

  • 石膏线：石膏线是房屋装修材料，主要作为室内原始棚顶阴角装饰，吊棚造型，墙面背景等装饰用途。
  • 石膏腻子：采用品诺筑家专供的龙牌石膏和腻子，粉刷石膏易于成型，强度高，不易收缩；嵌缝石膏易填充，适用局部修补，表面收光；腻子粉用于基层精找平，白度高易打磨。
  • 乳胶漆：又称为合成树脂乳液涂料，是有机涂料的一种，是以合成树脂乳液为基料加入颜料、填料及各种助剂配制而成的一类水性涂料。依照特点及适用范围，分为内墙乳胶漆、外墙乳胶漆、防水乳胶漆等。目前我们所使用世界十大品牌之一的立邦金装五合一乳胶漆，经全新增白配方设计的一款优质室内经济型装饰涂料，具有优异的透气性，干遮盖力，干白度，附着力和防霉性等多重功效，对墙面起到保护和美化的作用，是室内装饰墙面和天花板的理想选择。
  • 规范要求：网格带，使用降能低线槽开裂风险。
  • 规范要求：线槽进行网格带处理，降低线槽开裂风险，所有线槽都已挂网。
  • 规范要求：石膏板钉眼进行防锈处理，对石膏板自攻螺丝处点补防锈漆，经检查，未存在漏点。
  • 规范要求：对阴阳角放置阴阳角线，确保阴阳角顺直。
  • 绷带纸：降低石膏板开裂几率。
  • 规范要求：对石膏板接缝处进行防开裂处理，填补石膏粉，粘贴绷带纸，降低石膏板开裂几率。
  • 规范要求：对墙面打磨处理，砂纸目数为不低于240目，打磨完成后，项目经理进行平整度检查。
  • 防裂神器：拐角处增加防裂神器，降低石膏板拐角开裂几率。
  • 经检查，墙面无明显打磨痕迹。
  • 成品保护：墙漆前处理，对于墙面粉尘要清扫干净，所有需要分色面已用美纹纸进行保护，如开关底盒，门套木作等。
  • 规范要求：两遍面漆，采用晨阳水漆，已涂刷两遍，未发现透底和漏刷，流坠现象。
  • 规范要求：墙面冲筋是参考红外线水平仪找出墙面最高点，以此为基准等距在墙面冲出垂直石膏条筋，干透后批刮粉刷石膏，再用刮杆把墙面刮平。
  • 规范要求：世界十大品牌之一的上海加拿大原装进口喜加乳胶漆，产品采用ECO无味环保技术，不人为主动添加甲醛、苯、重金属、APEO等有害物质，净味。分型号不同，基本上底漆一遍，面漆一遍，完工涂刷均匀饱满、无明显色差、无刷痕。

【乳胶漆验收】

  • 顶面：目测无波浪、坑包，乳胶漆涂刷均匀无色差。
  • 石膏线、板线造型：板线上下口宽窄一致无弯曲，乳胶漆涂刷均匀无露底、无流坠。
  • 规范要求：隐藏处是否有腻子挂白（衣柜背后、分水器、窗台石下方）
  • 墙面：踢脚线、柜子处、门套线处.做垂直处理，误差3mm之内，其他地方顺平工艺，误差5mm之内
  • 腻子刮白到底2cm以下
  • 墙面壁纸、硅藻泥打磨：大白基层不露底（目测）。
  • 乳胶漆：有色涂料涂刷均匀饱满、无明显色差、无刷痕。

  • A：安装过程验收

  • 合页位置：每扇门需用3片合页，上面合页上口距门头约为240mm，中间合页下口距门头约840mm，下面合页下口距门底约为280mm。

  • B：安装结束验收
  • 规范要求：门槽口对角线长度差，允许偏差2（mm）
  • 规范要求：门框的正、侧面垂直度，允许偏差2（mm）
  • 规范要求：门扇与上框间留缝，允许偏差2（mm）
  • 规范要求：门套线接缝高低差，允许偏差2（mm）

【衣柜安装】

  • 准备阶段
  • 材料到场：板材运送到现场时一般都有纸板包装。拆开包装后验收一下各块板材和五金件。
  • 固定螺丝：板材上一般都按设计图在加工时预留好了钉眼，在现场的工人用专用工具把螺丝固定好。
  • 安装阶段
  • 固定框架：按照板材上的凹槽安装好衣柜的侧板和背板，调整各块板之间的位置。
  • 安装顶板：现场最好准备一个攀高用的凳子，顶板要举高了放置上去的，然后固定牢固。
  • 安装搁板：把搁板放置到事先敲好螺丝的地方，然后固定。注意调整搁板与背板、侧板的固定位置。
  • 固定门轨：在衣柜的顶边和底边都安装好门轨，门轨选合金的材料比较好，又轻又牢固。
  • 安装衣杆：在预留的位置打好固定螺丝，安装好衣杆。注意两边要平衡。
  • 收尾阶段
  • 安装门扇：两扇移门要贴和导轨的位置安装，先固定好门扇再把底边的导轨固定好，这样避免以后发生门扇脱落的现象。
  • 调整改良：全部安装完之后，对衣柜做个“全面检查”，有偏差的话及时调整。

【卫浴验收】

  • 洗漱盆安装注意
  • 规范要求：洗漱盆与排水管连接后应牢固密实，且便于拆卸，连接处不得敞口。洗漱盆与墙面接触部应用硅膏嵌缝。

  • 规范要求：给水管安装角阀高度一般距地面至角阀中心为250mm，如安装连体坐便器应根据坐便器进水口离地高度而定，但不小于100mm，给水管角阀中心一般在污水管中心左侧100mm或根据坐便器实际尺寸定位。
  • 规范要求：带水箱及连体马桶其水箱后背部离墙应不大于20mm。

  • 规范要求：开封后，经一一检查，无损坏。产品型号、色彩与合同上标注一致。
  • 规范要求：五金、压条型号，色彩与合同上标注一致。
